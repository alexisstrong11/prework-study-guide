HTML

  • Uses elements to create the structure (framework) of a webpage.
  • The head element contains information about the webpage.
  • Head Elements: meta, link, title, style, script and base.
  • Other Elements: Text, Image, Header, List, Meta-related, Content division, and Semantic.
  • The body element (semantic): represents the visible content shown to the user.
  • Attributes: define additional information about an element.
  • Declarations(not case-sensitive): tell browser what document type it should expect when opened.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds modifications in current working branch. "-A" flag indicates add all changes.
  • git commit -m: include commit message that describes what the commit contains
  • git pull origin main: Make local branch in sync with base branch in GitHub. "Pull" receives a branch's modifications into the local environment. "Origin" indicates source of the pull will be in the GitHub repo. "Main" indicates the branch.
  • Always do a "git pull" to ensure you have latest version of the project on your local computer before you start to work.
  • git push: Push changes we have locally to our remote GitHub branch.
  • git Terms: Repository, Branch, Markdown, Commit Changes, Pull Request, Merge, Issues
  • git Cheat Sheet: https://education.github.com/git-cheat-sheet-education.pdf
